# PHẦN 2: BÀI TOÁN DỰ BÁO LÀ GÌ?

## 2.1 Dự báo - Khái niệm đời thường

**Ví dụ 1: Dự báo thời tiết**
- Bạn nhìn ra cửa sổ thấy mây đen kéo đến
- Bạn đoán: "Chắc sắp mưa"
- Đây là dự báo đơn giản dựa trên kinh nghiệm

**Ví dụ 2: Dự báo kẹt xe**
- 7h sáng thứ Hai, bạn biết đường sẽ kẹt
- Vì bạn đã quan sát nhiều lần: "Thứ Hai luôn kẹt"
- Đây là nhận dạng pattern (mẫu hình)

**Ví dụ 3: Dự báo lượng khách quán cà phê**
- Chủ quán nhận thấy: Sáng sớm và chiều muộn đông khách
- Cuối tuần đông hơn ngày thường
- Quán đặt số lượng bánh dựa trên pattern này

## 2.2 Dự báo trong khoa học máy tính

```
┌─────────────────────────────────────────────────────────────────────────┐
│              DỰ BÁO = DỰ ĐOÁN TƯƠNG LAI DỰA TRÊN QUÁ KHỨ              │
└─────────────────────────────────────────────────────────────────────────┘

     QUÁ KHỨ                              TƯƠNG LAI
    ┌───────┐                            ┌───────┐
    │ Data  │ ──────────────────────────▶│ Model │ ───▶ Prediction
    │ t-12  │                            │  AI   │
    │ t-11  │                            │       │
    │ ...   │                            │       │
    │ t-1   │                            │       │
    │ t     │                            │       │
    └───────┘                            └───────┘
```

**Định nghĩa toán học đơn giản:**

Chúng ta có chuỗi số liệu theo thời gian:
- `y_t` = số máy bay tại thời điểm t
- `y_{t-1}` = số máy bay 5 phút trước
- `y_{t-2}` = số máy bay 10 phút trước

**Mục tiêu:** Tìm một công thức để tính `y_{t+k}` (k bước trong tương lai) từ các giá trị trước đó.

## 2.3 Tại sao dự báo khó?

```
┌─────────────────────────────────────────────────────────────────────────┐
│                     TẠI SAO DỰ BÁO KHÓ?                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  1. QUÁ KHỨ KHÔNG HOÀN TOÀN LẶP LẠI                                   │
│     - Hôm qua 8h sáng có 5 máy bay                                     │
│     - Hôm nay 8h sáng có 8 máy bay                                      │
│     - Lý do: Chuyến bay thay đổi theo ngày                             │
│                                                                          │
│  2. CÓ NHIỀU YẾU TỐ ẢNH HƯỞNG                                         │
│     - Thời tiết: Mưa → máy bay phải chờ                                │
│     - Sự kiện: Bamboo Airways thêm chuyến mới                          │
│     - Mùa: Tết → nhiều người đi máy bay                                │
│     - Thời gian: Giờ cao điểm đông hơn                                │
│                                                                          │
│  3. SỰ BẤT ĐỊNH (UNCERTAINTY)                                          │
│     - Chúng ta không thể đoán CHÍNH XÁC bao nhiêu máy bay              │
│     - Chỉ có thể nói: "Có thể từ 4 đến 8 máy bay"                   │
│     - Đây là "khoảng tin cậy" (prediction interval)                    │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

# PHẦN 3: TẠI SAO KHÔNG DÙNG CÔNG THỨC TOÁN ĐƠN GIẢN?

## 3.1 Phương pháp đơn giản nhất: Trung bình

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    PHƯƠNG PHÁP 1: TRUNG BÌNH                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  Công thức:  ŷ = (y₁ + y₂ + ... + y₁₂) / 12                          │
│                                                                          │
│  Ví dụ:                                                                  │
│  5, 7, 6, 5, 7, 8, 6, 5, 7, 6, 5, 7 → Trung bình = 6                 │
│                                                                          │
│  ⚠️ VẤN ĐỀ:                                                           │
│  - Tất cả các ngày đều cho ra kết quả 6                               │
│  - Không phân biệt 8h sáng (đông) vs 3h chiều (vắng)                 │
│  - Không quan tâm thời tiết                                             │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────┐
│                    PHƯƠNG PHÁP 2: TRUNG BÌNH ĐỘNG                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  Công thức:  ŷ = (y_{t-1} + y_{t-2} + ... + y_{t-n}) / n             │
│                                                                          │
│  Ví dụ (n=3):                                                          │
│  y_{t-1}=5, y_{t-2}=7, y_{t-3}=6                                      │
│  → ŷ = (5 + 7 + 6) / 3 = 6                                             │
│                                                                          │
│  ⚠️ VẤN ĐỀ:                                                           │
│  - Tất cả các giá trị đều có "trọng số" bằng nhau                    │
│  - Giá trị 5 phút trước quan trọng như giá trị 55 phút trước         │
│  - Không phản ánh xu hướng gần đây                                     │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

## 3.2 Phương pháp ARIMA - Phức tạp hơn một chút

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         ARIMA LÀ GÌ?                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  ARIMA = Auto-Regressive Integrated Moving Average                       │
│                                                                          │
│  Ý tưởng cơ bản:                                                        │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                  │   │
│  │  y_t = a₁·y_{t-1} + a₂·y_{t-2} + ... + b₁·ε_{t-1} + c        │   │
│  │                                                                  │   │
│  │  Trong đó:                                                       │   │
│  │  • y_{t-1}, y_{t-2} = giá trị quá khứ (Auto-Regressive)       │   │
│  │  • ε_{t-1} = sai số (Moving Average)                            │   │
│  │  • a₁, a₂, b₁, c = các hệ số được "học" từ dữ liệu           │   │
│  │                                                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                          │
│  Ví dụ đơn giản hóa:                                                   │
│  Số máy bay = 0.7 × (Số máy bay 5 phút trước) + 0.3 × (Số máy bay    │
│               10 phút trước) + 2                                        │
│                                                                          │
│  ⚠️ HẠN CHẾ LỚN NHẤT CỦA ARIMA:                                        │
│                                                                          │
│  1. GIẢ ĐỊNH QUAN HỆ TUYẾN TÍNH (Linear)                               │
│     - Thực tế: 10 máy bay ≠ 2 × 5 máy bay                             │
│     - Non-linear patterns: Giờ cao điểm đông GẤP 3 LẦN               │
│                                                                          │
│  2. CHỈ DÙNG MỘT BIẾN (Univariate)                                     │
│     - ARIMA cơ bản không dùng được với thời tiết                      │
│     - Muốn dùng thời tiết → phải mở rộng phức tạp (ARIMAX)          │
│                                                                          │
│  3. SEASONALITY PHẢI ĐỊNH NGHĨA TAY                                    │
│     - Muốn pattern theo giờ → phải tự cấu hình                        │
│     - Không tự động phát hiện                                           │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

## 4.2 Neural Network hoạt động như thế nào?

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    CÁCH NEURAL NETWORK "HỌC"                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  BƯỚC 1: KHỞI TẠO                                                      │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                  │   │
│  │   Bắt đầu với trọng số (weights) NGẪU NHIÊN                    │   │
│  │                                                                  │   │
│  │   Input: [5, 0.3, 80]  →  Output dự đoán: ???                   │   │
│  │                                                                  │   │
│  │   Đầu ra sai? Đương nhiên! Model mới bắt đầu mà.               │   │
│  │                                                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                          │
│  BƯỚC 2: TÍNH SAI SỐ (Error)                                           │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                  │   │
│  │   Đầu ra thực tế: 6 máy bay                                    │   │
│  │   Đầu ra dự đoán: 8 máy bay                                    │   │
│  │   ─────────────────────────────                                │   │

│  │                                                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                          │
│  BƯỚC 3: ĐIỀU CHỈNH TRỌNG SỐ (Backpropagation)                        │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                  │   │
│  │   "Nếu dự đoán cao hơn thực tế → giảm trọng số"               │   │
│  │   "Nếu dự đoán thấp hơn thực tế → tăng trọng số"              │   │
```

# PHẦN 10: CÁC THUẬT NGỮ VÀ GIẢI THÍCH

## 10.1 Glossary A-Z

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         GLOSSARY - THUẬT NGỮ                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  A                                                                       │
│  ────────────────────────────────────────────────────────────────────   │
│                                                                          │
│  AR-Net (Autoregressive Neural Network)                                  │
│  → Neural network đặc biệt dùng các giá trị QUÁ KHỨ để dự đoán         │
│    TƯƠNG LAI. Khác với neural network thường, AR-Net "nhìn" vào        │
│    quá khứ và tự hồi quy.                                               │
│                                                                          │
│  Autoregressive                                                         │
│  → "Tự hồi quy" - dùng chính các giá trị trước đó để dự đoán         │
│    giá trị tiếp theo. Như "nếu 5 phút trước có 7 máy bay,             │
│    5 phút sau có thể cũng gần 7".                                      │
│                                                                          │
│  ────────────────────────────────────────────────────────────────────   │
│  B                                                                       │
│  ────────────────────────────────────────────────────────────────────   │
│                                                                          │
│  Backpropagation                                                        │
│  → Thuật toán để neural network "học". Nó tính sai số và điều chỉnh    │
│    trọng số ngược từ output về input (backwards).                       │
│                                                                          │
│  Buffer Zone                                                            │
│  → Vùng đệm xung quanh sector chính. Nếu máy bay trong buffer zone     │
│    đang di chuyển vào trong sector, nó được tính là "inbound threat".  │
│                                                                          │
│  ────────────────────────────────────────────────────────────────────   │
│  E                                                                       │
│  ────────────────────────────────────────────────────────────────────   │
│                                                                          │
│  Epoch                                                                  │
│  → Một lần "học" toàn bộ dataset. Neural network học bằng cách         │
│    lặp qua data nhiều lần (nhiều epochs).                               │
│                                                                          │
│  Exogenous Variable                                                     │
│  → Biến ngoại sinh - yếu tố BÊN NGOÀI ảnh hưởng đến kết quả.           │
│    Ví dụ: thời tiết là exogenous variable cho số chuyến bay.          │
│                                                                          │
│  ────────────────────────────────────────────────────────────────────   │
│  F                                                                       │
│  ────────────────────────────────────────────────────────────────────   │
│                                                                          │
│  Forecast Horizon                                                       │
│  → Độ dài thời gian dự đoán. Nếu horizon = 30 phút, model dự đoán     │
│    "30 phút tới sẽ như thế nào".                                        │
│                                                                          │
│  Fourier Terms                                                          │
│  → Cách toán học để biểu diễn seasonality. Dùng sin/cos để tạo         │
│    ra các đường lặp lại (patterns).                                     │
│                                                                          │
│  ────────────────────────────────────────────────────────────────────   │
│  G                                                                       │
│  ────────────────────────────────────────────────────────────────────   │
│                                                                          │
│  Gradient Descent                                                       │
│  → Thuật toán tối ưu. "Đi xuống dốc" để tìm trọng số tốt nhất.       │
│    Like "tìm đường xuống núi trong sương mù".                          │
│                                                                          │
│  ────────────────────────────────────────────────────────────────────   │
│  L                                                                       │
│  ────────────────────────────────────────────────────────────────────   │
│                                                                          │
│  Lagged Regressor                                                       │
│  → Biến trễ - giá trị QUÁ KHỨ của một biến được dùng làm input.      │
│    Ví dụ: buffer_n (số máy bay từ North 5 phút trước).                 │
│                                                                          │
│  Layer (Tầng)                                                          │
│  → Tập hợp các neurons trong neural network. Nhiều layers =           │
│    "sâu" hơn (deep learning).                                           │
│                                                                          │
│  ────────────────────────────────────────────────────────────────────   │
│  N                                                                       │
│  ────────────────────────────────────────────────────────────────────   │
│                                                                          │
│  Neuron (Nơ-ron)                                                        │
│  → Đơn vị tính toán cơ bản trong neural network. Nhận input →         │
│    xử lý → tạo output. Like "tế bào não nhân tạo".                    │
│                                                                          │
│  Non-linear (Phi tuyến tính)                                            │
│  → Không theo đường thẳng. Thực tế phức tạp hơn, như                 │
│    đường cong thay vì đường thẳng.                                     │
│                                                                          │
│  ────────────────────────────────────────────────────────────────────   │
│  O                                                                       │
│  ────────────────────────────────────────────────────────────────────   │
│                                                                          │
│  Overfitting                                                            │
│  → Model "học vẹt" - nhớ training data thay vì học pattern.           │
│    Khi gặp data mới, nó sai. Like "học thuộc lòng" thay vì "hiểu".    │
│                                                                          │
│  ────────────────────────────────────────────────────────────────────   │
│  Q                                                                       │
│  ────────────────────────────────────────────────────────────────────   │
│                                                                          │
│  Quantile                                                               │
│  → Phân vị. 10th percentile = giá trị mà 10% data nằm dưới.           │
│    Giúp ước lượng uncertainty (khoảng dao động).                       │
│                                                                          │
│  ────────────────────────────────────────────────────────────────────   │
│  R                                                                       │
│  ────────────────────────────────────────────────────────────────────   │
│                                                                          │
│  Regressor                                                              │
│  → Biến đầu vào ảnh hưởng đến output. Weather, buffer là regressors.   │
│                                                                          │
│  Resampling                                                             │
│  → Chuyển data về cùng một tần suất đều đặn.                           │
│    Data 10 phút → resample về 5 phút bằng interpolation.              │
│                                                                          │
│  ────────────────────────────────────────────────────────────────────   │
│  S                                                                       │
│  ────────────────────────────────────────────────────────────────────   │
│                                                                          │
│  Seasonality                                                            │
│  → Tính chu kỳ. Pattern LẶP LẠI theo ngày/tuần/năm.                   │
│    Ví dụ: "8h sáng luôn đông hơn 3h chiều".                            │
│                                                                          │
│  ────────────────────────────────────────────────────────────────────   │
│  U                                                                       │
│  ────────────────────────────────────────────────────────────────────   │
│                                                                          │
│  Underfitting                                                           │
│  → Model quá đơn giản, không học được pattern.                         │
│    Like "không hiểu bài" thay vì "học vẹt".                            │
│                                                                          │
│  ────────────────────────────────────────────────────────────────────   │
│  W                                                                       │
│  ────────────────────────────────────────────────────────────────────   │
│                                                                          │
│  Weights (Trọng số)                                                     │
│  → Các con số quyết định mức độ ảnh hưởng của input lên output.       │
│    Model "học" = điều chỉnh weights để sai số nhỏ nhất.              │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

## 11.3 Câu hỏi thường gặp từ giám khảo

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    CÂU HỎI THƯỜNG GẶP TỪ THẦY                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  Q1: "Tại sao dùng NeuralProphet mà không dùng ARIMA?"                │
│  ────────────────────────────────────────────────────────────────────   │
│  A: ARIMA chỉ học được quan hệ TUYẾN TÍNH, trong khi air traffic      │
│     có pattern NON-LINEAR (giờ cao điểm tăng gấp 3 lần).              │
│     NeuralProphet với AR-Net học được cả linear và non-linear.        │
│     Thêm vào đó, NeuralProphet hỗ trợ native cho exogenous variables  │
│     (weather, buffer) và uncertainty estimation - ARIMA không có.       │
│                                                                          │
│  Q2: "Tại sao n_lags=12 mà không phải 6 hay 24?"                       │
│  ────────────────────────────────────────────────────────────────────   │
│  A: n_lags=12 nghĩa là lookback 60 phút (12 × 5 phút).                │
│     60 phút đủ để capture intra-hour patterns (arrival/departure).     │
│     6 lags (30 phút) có thể không đủ, 24 lags (2 giờ) cần nhiều      │
│     data hơn và dễ overfit. Đây là sweet spot cho dataset ~20-36 điểm.│
│                                                                          │
│  Q3: "Tại sao ar_layers=[32,32] mà không phải [16] hay [64,64]?"       │
│  ────────────────────────────────────────────────────────────────────   │
│  A: [16] có thể underfit - không đủ capacity để học air traffic       │
│     patterns phức tạp. [64,64] có thể overfit - quá nhiều parameters  │
│     so với data (~2100 params cho ~30 điểm data). [32,32] là sweet    │
│     spot balance giữa capacity và regularization.                     │
│                                                                          │
│  Q4: "Sao không dùng LSTM? Đó là state-of-the-art mà?"                │
│  ────────────────────────────────────────────────────────────────────   │
│  A: LSTM cần ~1000+ điểm data để hoạt động tốt. Dataset của chúng     │
│     tôi chỉ có ~30 điểm. Với data nhỏ như vậy, LSTM sẽ OVERFIT        │
│     được thiết kế đặc biệt cho small data scenarios.                   │
│                                                                          │
│  Q3: "Tại sao chỉ dùng 2 lớp ẩn (Hidden Layers)? Từng lớp làm gì?"      │
│                                                                          │
│  A: 2 lớp ẩn là "Điểm ngọt" (Sweet Spot) chống Overfitting trên dữ liệu│
│     nhiễu (Noise) của hàng không:                                        │
│     - Lớp 1 (Low-level): Tìm kiếm các "Góc nhọn thời gian", ví dụ nhận │
│       diện ra các đỉnh tăng vọt cục bộ hoặc các đường dốc ngắn hạn.     │
│     - Lớp 2 (High-level): Nhận diện "Hình dáng vĩ mô", tổng hợp tín hiệu│
│       từ Lớp 1 để phân biệt đâu là Tín hiệu thật (Xu hướng thực tế) và  │
│       đâu là Nhiễu (cơn bão ngẫu nhiên làm máy bay tăng vọt rồi giảm).  │
│                                                                          │
│  Q5: "Khoảng dao động 80% (quantiles [0.1, 0.9]) lấy ở đâu?"          │
│  ────────────────────────────────────────────────────────────────────   │
│  A: 80% interval là sweet spot cho air traffic management:             │
│     - 50% interval [0.25, 0.75] quá hẹp, không bao phủ đủ            │
│     - 95% interval [0.025, 0.975] quá rộng, không actionable           │
│     - 80% interval vừa đủ rộng để coverage tốt, vừa đủ hẹp để        │
│       controller có thể plan dựa trên nó.                              │
│                                                                          │
│  Q6: "Làm sao model biết weather ảnh hưởng thế nào?"                  │
│  ────────────────────────────────────────────────────────────────────   │
│  A: Weather được thêm vào như "future regressors". Trong quá trình     │
│     training, model học các hệ số (coefficients) cho mỗi weather      │
│     variable. Ví dụ: precipitation coefficient = -0.5 nghĩa là         │
│     "mỗi mm mưa tăng, số máy bay giảm 0.5". Model tự học mối          │
│     quan hệ này từ data, không cần chỉ định thủ công.                 │
│                                                                          │
│  Q7: "Buffer count là gì, tại sao nó giúp dự đoán?"                   │
│  ────────────────────────────────────────────────────────────────────   │
│  A: Buffer zones là các vùng xung quanh sector chính. Khi máy bay    │
│     trong buffer zone di chuyển VÀO sector, nó sẽ làm tăng count.     │
│     buffer_n/s/e/w = số máy bay đang tiến vào từ 4 hướng.             │
│     Nếu buffer count cao → có nhiều máy bay sẽ vào sector →           │
│     dự đoán count tương lai cao hơn. Đây là PREDICTIVE INFORMATION.   │
│                                                                          │
│  Q8: "Model có thể predict khi nào DỰNG ĐOÁN không?"                   │
│  ────────────────────────────────────────────────────────────────────   │
│  A: Khoảng dao động (prediction interval) chính là để handle cái này.  │
│     Nếu prediction = 6 (khoảng 4-8), thực tế có thể 3 hay 9.         │
│     Controller được training để hiểu: "Dự đoán 6 nhưng có thể 8"     │
│     → Cần prepare cho scenario xấu nhất (8 máy bay).                    │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

```
    │   ├─ Calculate speed, heading

    │   └─ Check if intersects core sector
    ├─ Count: buffer_n, buffer_s, buffer_e, buffer_w
    │
    ▼
[4] DATABASE STORAGE (SQLite)
    │
    └─ Table: sector_density
        ├─ timestamp
        ├─ aircraft_count
        ├─ precipitation, wind_speed, cloud_cover, visibility
        └─ buffer_n, buffer_s, buffer_e, buffer_w
    │
    ▼ (every 10 minutes)
[5] PREPROCESSING (app.py)
    │
    ├─ Load from SQLite
    ├─ Rename: timestamp → ds, aircraft_count → y

    │
```

### B. Mathematical Formulation

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    MATHEMATICAL FORMULATION                             │
└─────────────────────────────────────────────────────────────────────────┘

OBSERVATION MODEL:

  y_t = f(y_{t-1}, y_{t-2}, ..., y_{t-12};
           x^{weather}_t;
           z^{buffer}_{t-1}, ..., z^{buffer}_{t-12}) + ε_t

WHERE:
  y_t         = aircraft count at time t (target variable)
  y_{t-k}     = lagged target values (autoregressive)
  x^{weather}_t = [precipitation, wind_speed, cloud_cover, visibility]
  z^{buffer}_t = [buffer_n, buffer_s, buffer_e, buffer_w]
  ε_t         = noise term
  f()         = neural network function (AR-Net)

NEURAL NETWORK ARCHITECTURE:

  h_1 = ReLU(W_1 · [y_{t-12:t}, x_t, z_{t-12:t}] + b_1)
  h_2 = ReLU(W_2 · h_1 + b_2)
  ŷ_{t+1:t+6} = W_3 · h_2 + b_3

LOSS FUNCTION (Quantile Regression):

  L(θ) = Σ_{q∈{0.1,0.5,0.9}} Σ ρ_q(y_i - ŷ_q,i)

  WHERE:
    ρ_q(u) = u·(q - I(u<0))   [pinball loss]
    q = 0.1, 0.5, 0.9 (quantiles)

PREDICTION INTERVAL:

  P(y ∈ [ŷ_10, ŷ_90]) ≈ 0.80

  I.e., 80% confidence that actual value lies within interval
```
